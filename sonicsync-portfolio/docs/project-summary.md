## Problem addressed

Playing the same audio on several smartphones normally produces unsynchronised output because each device starts, buffers and processes audio independently. SonicSync addressed this problem by proposing a master-client mobile system in which one Android phone controls playback and other phones on the same local Wi-Fi network operate as wireless speakers.

## Proposed solution

The system used a master audio-source device and one or more connected client devices. The master device was responsible for selecting audio, controlling playback and coordinating connected devices. Client devices received the relevant control or timing information and reproduced the selected audio as closely as possible to the master's start time.

## Objectives

1. Turn Android smartphones into a wireless multi-speaker audio system.
2. Connect additional phones to a master audio-source device over a local network.
3. Support simultaneous playback on participating devices.
4. Use a straightforward Android interface for discovery, connection and control.
