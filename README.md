# AI-Assisted Mobile Phishing Awareness Research Demo

This repository is a supporting technical evidence package for a proposed Master by Research/MPhil application in Computer Science / Cybersecurity.

## Proposed research title

**AI-Assisted Mobile Phishing Awareness and Detection: A Quantitative Evaluation of Microlearning Warnings for University Students**

## Purpose of this repository

This repository is not a finished software product and it is not a private university application file. It is a small, public research demo showing that I can:

- understand a real cybersecurity problem;
- work with URL-based phishing detection features;
- design a measurable quantitative research study;
- connect technical detection with human-centred cybersecurity awareness;
- prepare reproducible research materials for supervisor discussion.

## Research problem

Mobile users receive phishing links through SMS, email, messaging apps, social media and browser notifications. Many users struggle to recognise suspicious domains, fake login pages, urgent wording, shortened links and misleading security messages. Existing technical detection tools can flag risky links, but users may still ignore warnings if the warning is unclear, too technical or poorly designed.

The proposed research investigates whether **simple mobile microlearning warning cards**, supported by AI-assisted phishing risk cues, can improve user awareness and reduce risky link-clicking decisions.

## Planned research method

The proposed research can be completed realistically through:

1. A literature review on phishing, mobile security warnings and human-centred cybersecurity.
2. A small phishing URL detection demo using public datasets or controlled sample data.
3. A microlearning warning-card prototype designed for mobile users.
4. A quantitative pre-test/post-test study with university students.
5. Statistical analysis of awareness scores, detection accuracy, user confidence and perceived usefulness.

## Possible quantitative measures

- phishing awareness score before and after microlearning warnings;
- correct identification of legitimate vs suspicious links;
- perceived confidence in detecting phishing attempts;
- warning clarity and usefulness ratings;
- model evaluation metrics such as accuracy, precision, recall and F1-score.
